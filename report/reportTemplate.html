Report of Auto discovered problematic event processing


Introduction:


An approach of reading historic data from Netcool reporter database and auto classify the alarms based on event processing status resulted previously.

Know your environment - Using Machine learning clustering algorithm historic alarms will be grouped based on the respective event processing status.

Identify failures - Perform sentiment analysis over the event processing status updates the respective results at alert fields which enables to identify problematic areas.

Descion rule of thumb :-


1 - Given alarm contains negative sentiment result for its any of the event processing states is considered as faulty event processing.

2 - Given alarm contains neutral sentiment result for half and more than half of the input status fields.

3 - Cluter numbered -1 are called as outlier i.e new combination of issue is getting reported at env.

Missed event processing can be predected early by which required actions to address the event processing needs can be preformed (eg: Log collection for troubleshooting,Perform event processing manually to avoid busniess impact
